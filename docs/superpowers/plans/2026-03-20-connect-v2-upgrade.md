# Connect V1 → V2 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade all TypeScript Connect RPC packages from V1 (`^1.4.0`) to V2 (`^2.0.0`) and `@bufbuild/protobuf` from `^1.9.0` to `^2.2.0` across the Cosmo platform.

**Architecture:** Incremental upgrade starting with code generation and the `connect` package (foundation), then propagating changes outward to consumers: shared → controlplane → cli → studio → protographic. Each phase produces a buildable, testable state.

**Tech Stack:** `@connectrpc/connect` v2, `@bufbuild/protobuf` v2, `protoc-gen-es` v2, `@connectrpc/connect-query` v2, `@connectrpc/connect-web` v2, `@connectrpc/connect-node` v2, `@connectrpc/connect-fastify` v2

---

## Migration Reference

Key API changes (from [MIGRATING.md](https://github.com/connectrpc/connect-es/blob/main/MIGRATING.md)):

| V1 | V2 |
|----|-----|
| `new Message({...})` | `create(MessageSchema, {...})` |
| `msg.toBinary()` | `toBinary(MessageSchema, msg)` |
| `msg.toJson()` | `toJson(MessageSchema, msg)` |
| `msg.toJsonString()` | `toJsonString(MessageSchema, msg)` |
| `Message.fromBinary(bytes)` | `fromBinary(MessageSchema, bytes)` |
| `Message.fromJson(json)` | `fromJson(MessageSchema, json)` |
| `Message.fromJsonString(str)` | `fromJsonString(MessageSchema, str)` |
| `PlainMessage<T>` | Just `T` (messages are plain by default) |
| `PartialMessage<T>` | `MessageInitShape<typeof MessageSchema>` |
| `createPromiseClient(Service, transport)` | `createClient(Service, transport)` |
| `PromiseClient<typeof Service>` | `Client<typeof Service>` |
| `import { Timestamp } from "@bufbuild/protobuf"` | `import { TimestampSchema } from "@bufbuild/protobuf/wkt"` |
| `*_connect.ts` files (service descriptors) | Merged into `*_pb.ts` |
| `protoc-gen-connect-es` plugin | Removed (handled by `protoc-gen-es` v2) |
| `protoc-gen-connect-query` plugin | Replaced by `@connectrpc/protoc-gen-connect-query` v2 |
| `jsonOptions: { emitDefaultValues: true }` | `jsonOptions: { alwaysEmitImplicit: true }` |
| `credentials: 'include'` on transport | Use `fetch` override: `fetch: (input, init) => fetch(input, { ...init, credentials: "include" })` |
| `typeRegistry` | `registry` |

import { fromJson, fromJsonString } from '@bufbuild/protobuf';

export function routerConfigFromJson(json: JsonValue): RouterConfig {

}

## Risk Areas & Notes

1. **`PlainMessage<T>` removal** — The biggest mechanical change. 181 handler files in controlplane all use `PlainMessage<ResponseType>`. In V2, messages are plain objects by default, so the wrapper is unnecessary. However, verify that the handler return type signature still type-checks without it.

2. **`emitDefaultValues` → `alwaysEmitImplicit`** — The controlplane sets `emitDefaultValues: true` in `handlerOptions`. This option was renamed in V2. Getting this wrong could break JSON serialization for clients.

3. **`credentials: 'include'` removal** — Studio transport uses this directly. Must switch to `fetch` override pattern or cookies won't be sent.

4. **connect-query hooks** — Verify that `useQuery` and `useMutation` from `@connectrpc/connect-query` v2 have the same API. The import pattern from generated `*_connectquery.ts` files may change.

5. **Generated file naming** — After removing `protoc-gen-connect-es`, verify that `protoc-gen-es` v2 puts service descriptors in `*_pb.ts` files and that the connect-query plugin still generates `*_connectquery.ts` files with the expected names.

6. **Exposed types in `@wundergraph/cosmo-connect`** — This is a published npm package. The V2 upgrade changes the shape of all exported types (messages become plain). Any external consumers of this package will also need to upgrade `@bufbuild/protobuf` to v2.

7. **`ServiceImpl` type** — Used in `PlatformService.ts` and `NodeService.ts` as `Partial<ServiceImpl<typeof PlatformService>>`. V2 service descriptors have a different shape (now in `*_pb.ts`), which could affect the generic parameter. Verify this compiles.

8. **`Client` naming conflict in CLI** — `cli/src/core/client/client.ts` exports a local `interface Client`. Connect V2 renames `PromiseClient` → `Client`. Must alias the import to avoid collision.

9. **`ResponseMessage` type constraint** — `controlplane/src/core/util.ts` defines `handleError<T extends ResponseMessage>`. If `ResponseMessage` is defined using `PlainMessage`, it also needs updating.

10. **`emitDefaultValues` rename verification** — The plan assumes `emitDefaultValues` → `alwaysEmitImplicit`. Verify the exact V2 option name by checking `@bufbuild/protobuf` v2 type definitions before applying. Also update `shared/test/router.config.test.ts`.
